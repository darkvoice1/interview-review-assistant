chore(repo): 删除临时 commit 信息文件

改动背景：
1. 首次提交时为了按模板生成提交信息，临时创建了本地 commit 消息文件。
2. 该文件不属于项目源码，保留在仓库中会增加无效噪音。

本次改动：
1. 删除误加入仓库的 `.git-commit-message.txt` 临时文件。
2. 保持提交历史与仓库文件结构更干净。
3. 确保后续查看项目时不会把临时协作文件误认为正式资产。

测试补充：
1. 本次没有新增自动化测试。
2. 使用 `git status --short` 确认仅包含该临时文件删除变更。
3. 执行结果：删除提交完成，工作区恢复干净。

收益说明：
1. 仓库内容更聚焦于正式源码与文档。
2. 降低后续维护时的误读成本。
3. 为后续持续提交保持更整洁的提交基线。
